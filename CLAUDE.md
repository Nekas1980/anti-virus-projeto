## Overview

**anti-virus-projeto** is an educational malware detection tool built during an IEFP Cybersecurity Bootcamp. It uses SHA256 hashing for signature-based file scanning, with optional VirusTotal API enrichment. The project emphasizes clean architecture, comprehensive testing, and professional packaging.

**Repository**: https://github.com/Nekas1980/anti-virus-projeto

## Architecture at a Glance

**5-layer stack**:
1. **UI Layer**: CLI (Virus_project.py main), GUI (gui3.py with threading)
2. **Scanning Engine**: Virus_project.py — recursive directory traversal, SHA256 hashing, exclusion filtering
3. **Support Services**: scheduler.py (interval-based background tasks), virustotal_updater.py (API enrichment), report_generator.py (HTML/JSON export)
4. **Data Layer**: signatures.json (malware hashes), exclusions.json (skip patterns), schedule_config.json (scheduler config)
5. **Infrastructure**: Tests (18 unit tests), CI/CD (GitHub Actions matrix), Packaging (setup.py, PyInstaller)

**Key principle**: Single responsibility — each module has one clear role.

## Configuration

**signatures.json** (malware database):
```json
{
  "malware_hashes": {
    "abc123def...": "Trojan.Win32.Generic",
    "xyz789abc...": "Backdoor.Linux.SSH"
  }
}
```

**exclusions.json** (skip patterns — fnmatch style):
```json
{
  "exclusion_patterns": [
    "node_modules",
    ".git",
    "venv",
    "*.tmp"
  ]
}
```

**schedule_config.json** (auto-generated by scheduler):
- Edit manually or regenerate via `ScanScheduler().create_schedule_config()`
- Interval format: `{"interval": 3600, "enabled": true}` (seconds)

## Code Patterns to Know

### Scanning Pattern
```python
def scan_directory(path, signatures, exclusions):
    for root, dirs, files in os.walk(path):
        if should_skip_path(root, exclusions):
            continue
        for file in files:
            file_path = os.path.join(root, file)
            yield scan_file(file_path, signatures)
```

### Configuration Loading
```python
def load_signatures(path):
    with open(path, 'r') as f:
        config = json.load(f)
    return config.get('malware_hashes', {})
```

### GUI Threading (Avoid Freezing UI)
- `gui3.py` uses `threading.Thread` to run scans in background
- GUI remains responsive during long scans
- Results posted back via safe queue or callback pattern

## Quick Reference: File Flow

1. User launches `gui3.py` or runs `python Virus_project.py <dir>`
2. Core engine loads `signatures.json` + `exclusions.json`
3. `scan_directory()` recursively traverses with filtering
4. `scan_file()` computes SHA256 hash, checks against database
5. Results yielded as `ScanResult` objects (file_path, status, reason, hash)
6. Optional: `virustotal_updater` enriches unknown files via API
7. `report_generator` converts results to HTML/JSON in `output/`
8. Optional: `quarantine_file()` moves infected files to `quarantine/`
9. Optional: `scheduler.py` runs periodic scans on a schedule
